💌 AWS Serverless Contact Form API

Problem: Needed a scalable and secure way to handle contact form submissions from a static website and deliver messages via email without managing servers.

Solution: Built a serverless architecture using Amazon API Gateway and AWS Lambda (Python) to process form submissions, integrated with a static frontend hosted on Amazon S3 and delivered via Amazon CloudFront. Configured CORS for browser-based requests, implemented input validation, and integrated Amazon SES with a custom domain to send transactional emails.

Result: Delivered a fully serverless, production-style contact system capable of real-time email notifications using a custom domain, with no infrastructure to manage and aligned with AWS best practices for scalability, security, and reliability.

Secure Backend API (Serverless Authentication)

Problem: The contact form API was publicly accessible, allowing anyone to send requests—leading to potential spam, abuse, and unnecessary costs.

Solution: Implemented authentication using Amazon Cognito and secured the API with a JWT authorizer in Amazon API Gateway. Configured Cognito’s hosted login to issue JWT tokens and enforced token validation before requests reach AWS Lambda. Updated the frontend to include Bearer tokens in API requests and handled request validation and processing securely, maintaining email delivery via Amazon SES.

Result: Transformed a public API into a secure, authenticated backend where only authorized users can submit requests, preventing unauthorized access and abuse. Established a production-ready, identity-aware serverless architecture aligned with AWS security best practices.
